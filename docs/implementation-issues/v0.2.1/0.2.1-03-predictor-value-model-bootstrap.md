# Issue 0.2.1-03

Status: completed on current branch

## Title

Predictor/value-model bootstrap

## Objective

Start the predictor branch using `combat-outcomes.jsonl` that `v0.2` already exports.

## Problem

The repository already records useful combat outcome data, but it is not yet turned into a first-class derived dataset or a trainable predictor path. That leaves a high-value signal unused.

## In Scope

- Define a derived predictor dataset format from `combat-outcomes.jsonl`
- Add a preprocessing or extraction command that writes predictor-ready artifacts
- Add a baseline training command for a combat outcome or value predictor
- Emit evaluation summaries for predictor experiments
- Reserve a clean interface for later policy-time score consumption

## Out Of Scope

- Full integration of predictor scores into the active DQN policy
- Large model sweeps
- Cross-machine training workflows

## Dependencies

- Prefer `0.2.1-01` for cleaner run artifacts, but can start from existing `v0.2` exports

## Affected Areas

- `src/sts2_rl/data/`
- `src/sts2_rl/predict/` or equivalent new package
- `src/sts2_rl/cli.py`
- `artifacts/` and `data/` output conventions
- `tests/`

## Acceptance Criteria

- Predictor dataset extraction can be run from the CLI on existing session outputs
- A baseline predictor can be trained end-to-end from the extracted dataset
- Predictor training writes summaries and metrics into `artifacts/`
- Tests cover dataset extraction and predictor artifact generation

## Verification

- Automated tests for extraction logic and training entry points
- Local predictor smoke run using real exported combat outcomes

## Delivered

- `src/sts2_rl/predict/` package with feature extraction, dataset extraction, model save/load, scoring, and trainer modules
- `sts2-rl predict dataset extract` CLI command for recursive `combat-outcomes.jsonl` ingestion
- `sts2-rl predict train combat-outcome` CLI command that trains a baseline multi-head predictor and writes model, metrics, and summary artifacts
- Automated tests covering extracted dataset contents, model artifact generation, and runtime scoring interfaces

## Notes

This issue is intentionally early because it uses data the repo already owns and opens a parallel branch with high upside.
