# Issue 0.2.1-04

Status: completed on current branch

## Title

Heuristic collector strengthening

## Objective

Raise non-combat decision quality so collection data and run-level outcomes improve before heavier RL work.

## Problem

The current heuristics are functional and instrumented, but they are still shallow in route choice, reward handling, shop behavior, and rest-site logic. That caps run quality and weakens all downstream imitation or offline-RL datasets.

## In Scope

- Improve route selection heuristics
- Improve card reward and relic reward selection heuristics
- Improve shop purchase and removal heuristics
- Improve rest-site upgrade, heal, and other campfire choices
- Preserve or improve decision reason logging for each heuristic action
- Add focused unit coverage for new heuristic branches

## Out Of Scope

- Replacing heuristics with an LLM planner
- Full-run search methods
- Combat policy redesign beyond small heuristic glue improvements

## Dependencies

- Prefer `0.2.1-01` so heuristic improvements can be measured across complete runs

## Affected Areas

- `src/sts2_rl/collect/policy.py`
- `src/sts2_rl/collect/runner.py`
- `src/sts2_rl/data/trajectory.py`
- `tests/`

## Acceptance Criteria

- Heuristic policy covers the main non-combat screens seen in normal runs with explicit reasons
- Collection summaries expose stronger coverage and fewer no-op or fallback decisions
- Unit tests cover the added decision branches
- Live collection smoke runs complete with fewer avoidable stalls or obviously poor non-combat choices

## Verification

- Automated tests for route, reward, shop, and rest decisions
- Local collection smoke run and artifact review

## Delivered

- `src/sts2_rl/collect/policy.py` upgraded from shallow heuristics to a context-aware `simple-policy-v3`
- Improved route, event, reward-card, selection, shop, and rest-site decisions with explicit categorical `decision_reason` values
- Multilingual text and numeric scoring that works against current localized STS2 payloads instead of relying on English-only card text
- `summary.json` decision-stage and decision-reason histograms via the trajectory recorder for live coverage inspection
- Unit coverage for map routing, hybrid reward/card-selection handling, skip logic, card removal, shop removal, rest behavior, and game-over recovery

## Notes

This is not temporary glue. Strong heuristics improve evaluation quality, offline datasets, and the overall reliability of the system.
