# Issue 0.2.2-02

## Title

Replay and compare artifact hardening

## Status

Completed on the current branch.

## Objective

Move replay and checkpoint comparison from a boolean `prepare_main_menu` assumption to explicit anchored-start artifacts that explain how the runtime was normalized before evaluation.

## Problem

`v0.2.1` replay artifacts could tell whether repeated runs diverged, but they did not preserve enough information about how each iteration was prepared. That made it hard to distinguish real policy divergence from preparation drift.

## In Scope

- replace replay/compare preparation booleans with explicit `prepare_target`
- persist normalization reports into replay and compare iteration artifacts
- add summary fields that expose normalization stop reasons and strategy histograms
- keep backward-compatible CLI behavior for existing `--prepare-main-menu` flows

## Out Of Scope

- headless replay
- guaranteeing deterministic combat RNG
- adding a new evaluation transport

## Dependencies

- `0.2.2-01` Runtime normalization and anchored starts

## Affected Areas

- `src/sts2_rl/train/replay.py`
- `src/sts2_rl/train/compare.py`
- `src/sts2_rl/cli.py`
- `tests/test_train_replay.py`
- `tests/test_train_compare.py`

## Acceptance Criteria

- replay and comparison commands accept explicit preparation targets
- iteration payloads include `prepare_target` and `normalization_report`
- replay and comparison summaries expose normalization outcome histograms
- tests cover the updated artifact shape

## Verification

- automated replay and compare tests
- local replay / compare smoke can use `--prepare-target main_menu` or `--prepare-target character_select`

## Delivered

- replay suites now record per-iteration `prepare_target`, `normalization_report`, and anchored start payloads.
- checkpoint comparison iterations now record the same preparation metadata and surface normalization histograms in aggregate summaries.
- CLI commands now support `--prepare-target` while preserving `--prepare-main-menu` as a compatibility layer.
