# Issue 0.2.2-01

## Title

Runtime normalization and anchored starts

## Status

Completed on the current branch.

## Objective

Make live runtime recovery explicit and reusable so evaluation flows can start from known screens instead of assuming the game is already parked at the right boundary.

## Problem

`v0.2.1` could normalize some replay flows to `MAIN_MENU`, but that logic was narrow, replay-specific, and not visible to operators. Live runs still depended on the user manually recovering screens such as `REWARD`, `MODAL`, `TIMELINE`, or `GAME_OVER`.

## In Scope

- add a general runtime normalization helper and report format
- recover to `main_menu` or `character_select`
- prefer direct exit actions before falling back to heuristic progression
- add a CLI command that runs normalization over configured instances and writes artifacts

## Out Of Scope

- enforcing engine determinism
- savestates or simulator resets
- replacing the existing action bridge

## Dependencies

- `0.2.1-01` Episode lifecycle controller

## Affected Areas

- `src/sts2_rl/runtime/normalize.py`
- `src/sts2_rl/runtime/__init__.py`
- `src/sts2_rl/cli.py`
- `tests/test_runtime_normalize.py`

## Acceptance Criteria

- the repo can normalize a live instance to `main_menu` or `character_select`
- normalization emits machine-readable reports with stop reasons and action sequences
- direct exits such as `abandon_run` and modal dismissal are preferred over deeper heuristic traversal
- automated tests cover at least one abandon-run path and one menu-to-character-select path

## Verification

- automated tests for `normalize_runtime_state`
- CLI wiring that writes summary artifacts under `artifacts/runtime-normalize`

## Delivered

- `normalize_runtime_state(...)` now emits `RuntimeNormalizationReport` with action sequence, strategy histogram, and final observation.
- `sts2-rl instances normalize` now normalizes configured instances and writes per-instance JSON plus `normalize-summary.json`.
- tests now cover `REWARD -> abandon_run -> MODAL -> MAIN_MENU` and `MAIN_MENU -> CHARACTER_SELECT`.
