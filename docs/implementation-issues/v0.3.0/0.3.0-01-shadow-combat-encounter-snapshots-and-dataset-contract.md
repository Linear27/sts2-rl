# Issue 0.3.0-01

## Status

Completed on 2026-04-13.

## Title

Shadow combat encounter snapshots and dataset contract

## Objective

Materialize a first-class encounter-start dataset from live trajectory logs so local planner or simulator research can run against stable combat snapshots instead of scraping ad hoc JSONL.

## Problem

The repo can already collect full live trajectories and combat outcomes, but it still lacks a reusable artifact that says: "this is the exact combat start state, legal action set, strategic context, and real downstream outcome for one encounter." Without that contract, every shadow-environment experiment would start with one-off parsing and inconsistent assumptions.

## In Scope

- shadow encounter record schema and loader
- manifest-built `shadow_combat_encounters` dataset kind
- combat-start snapshot extraction from trajectory logs
- split-safe grouping by combat id
- CSV/JSONL exports and summary statistics for encounter snapshots
- config example and roadmap / issue documentation

## Out Of Scope

- a full combat simulator
- local roll-forward dynamics or search execution
- replacing the live runtime as the source of truth

## Dependencies

- trajectory recorder and combat boundary artifacts from `v0.2.x`
- strategic context fields from `v0.2.5`

## Affected Areas

- `src/sts2_rl/data/`
- `configs/datasets/`
- `docs/`
- `tests/`

## Acceptance Criteria

- the repo can build a dedicated encounter-start dataset from trajectory logs with no one-off scripts
- records include full start snapshots when available, plus action traces and real outcomes
- dataset manifests can split by combat id to avoid encounter leakage across splits
- summaries expose encounter-family, action-trace, and snapshot-coverage statistics
- tests cover manifest build, record extraction, summary generation, and loader behavior

## Implementation Breakdown

### 1. Encounter record contract

- add a typed shadow encounter record with stable filenames and load helpers
- persist start snapshot fingerprints, strategic context, and downstream combat outcome labels

### 2. Dataset integration

- add `shadow_combat_encounters` to the dataset manifest contract
- build encounter records directly from `combat_started`, combat `step`, and `combat_finished` trajectory records
- support `combat_id` split grouping for shadow datasets

### 3. Reporting

- export `encounters.jsonl` and `encounters.csv`
- report encounter-family histograms, action-trace counts, hp stats, and snapshot coverage

### 4. Documentation and examples

- add a shadow dataset example manifest
- update roadmap / issue index so the next shadow-environment work has an execution anchor

### 5. Tests

- cover encounter extraction with full snapshots and finished outcomes
- cover split grouping by `combat_id`
- cover dataset summary fields and record loader behavior

## Deliverables

- shadow encounter record schema and loader
- `shadow_combat_encounters` dataset build path
- combat-id split grouping support
- encounter summary / CSV exports
- config example and tests
