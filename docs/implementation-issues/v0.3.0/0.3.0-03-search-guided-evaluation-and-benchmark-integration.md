# Issue 0.3.0-03

## Status

Completed on 2026-04-13.

## Title

Search-guided evaluation and benchmark integration

## Objective

Connect shadow-search results back into the repo's main operational loop so benchmark compare cases, promotion gates, and registry summaries can reason about offline shadow evidence instead of treating it as a separate research-only artifact.

## Problem

`0.3.0-02` made shadow evaluation executable, but the result still sat outside the main evaluation stack. Live benchmark cases could compare policy packs and checkpoints, yet they could not attach shadow evidence from fixed encounter datasets, and promotion logic had no way to require that a candidate also improves logged-action alignment in the shadow harness. That left shadow research disconnected from the decisions the repo actually makes.

## In Scope

- benchmark manifest support for shadow-backed compare cases
- compare-case sidecar shadow artifacts
- promotion checks that can consume shadow compare metrics
- suite summaries and registry metrics that expose shadow compare coverage
- examples, issue docs, and regression tests

## Out Of Scope

- simulator-driven live runtime replacement
- checkpoint shadow evaluation for arbitrary model binaries
- registry-native standalone shadow report family support

## Dependencies

- `0.3.0-02` shadow compare harness
- existing benchmark compare and promotion pipeline
- registry inspection of benchmark suite and benchmark case summaries

## Affected Areas

- `src/sts2_rl/train/benchmark_suite.py`
- `src/sts2_rl/registry/core.py`
- `src/sts2_rl/cli.py`
- `configs/benchmarks/`
- `tests/`
- `docs/`

## Acceptance Criteria

- benchmark `compare` cases can attach a shadow source and emit sidecar shadow comparison artifacts
- policy-pack compare cases can feed shadow metrics into promotion checks without ad hoc scripts
- suite summaries expose aggregate shadow coverage and delta statistics
- registry benchmark entries surface shadow metrics for leaderboard and compare workflows
- tests cover manifest path resolution, shadow benchmark execution, promotion integration, and registry inspection

## Implementation Breakdown

### 1. Benchmark manifest support

- add compare-case shadow config with a resolved dataset source path
- validate that shadow benchmarking only runs for policy-pack compare cases

### 2. Shadow sidecar execution

- run `shadow combat-compare` automatically inside benchmark compare cases
- persist shadow summary and comparisons paths under the case artifact tree

### 3. Promotion integration

- extend promotion thresholds with optional shadow gates
- allow shadow comparable-count and delta metrics to participate in candidate promotion

### 4. Summary and registry integration

- expose per-case shadow metrics in benchmark case summaries
- aggregate suite-level shadow coverage and delta stats
- surface shadow metrics in registry snapshots for benchmark suites and benchmark cases

### 5. Docs and tests

- extend the example benchmark manifest with a shadow-backed policy compare
- update roadmap and README
- add tests for manifest resolution, benchmark execution, promotion, and registry inspection

## Deliverables

- shadow-aware compare-case benchmark manifest contract
- benchmark compare payloads with `shadow` and shadow artifact references
- promotion checks that can gate on shadow compare evidence
- registry snapshot support for shadow-backed benchmark summaries
- config example, docs, and regression tests
