# Issue 0.3.2-03

## Status

Complete on the current branch.

## Title

Public run normalization and derived strategic stats

## Objective

Normalize archived public runs into repo-native strategic summaries that can support route analysis, card / shop priors, and benchmark slicing without pretending public runs are step-level replay data.

## Problem

Once raw public runs exist locally, they are still too source-shaped for direct policy or report consumption. The repo needs a typed normalization layer that can summarize route progression, acts, encounters, shops, rewards, relics, and deck growth while preserving provenance. Without that layer, downstream code would either parse raw website payloads directly or misuse public runs as if they were local runtime trajectories.

## In Scope

- typed normalized public-run records
- derived aggregates for route, card, shop, event, relic, and encounter statistics
- build / version / character / ascension coverage summaries
- benchmark-slice exports from normalized public runs
- tests and documentation

## Out Of Scope

- direct online sync from external sources
- live decision-time consumption hooks
- step-level action imitation from public runs

## Dependencies

- `0.3.2-02` raw public run archive contract
- current benchmark, prior, and reporting summary conventions

## Affected Areas

- `src/sts2_rl/data/`
- `src/sts2_rl/train/`
- `tests/`
- `docs/`

## Acceptance Criteria

- the repo can normalize raw archived public runs into typed strategic records with explicit source provenance
- normalized outputs expose route, act, encounter, reward, shop, relic, and deck summaries that are stable across refreshes
- summaries report build coverage, character coverage, ascension distribution, completion / abandonment counts, and derivation coverage
- derived exports can be reused by later prior and benchmark consumers without reparsing raw archive payloads
- tests cover field normalization, missing-field tolerance, and representative derived aggregates

## Implementation Breakdown

### 1. Typed normalized contract

- map source-shaped run payloads into a repo-native normalized run schema
- preserve source ids, source URLs, archive session lineage, and build metadata

### 2. Derived strategic stats

- compute route and room-type frequencies
- derive card / shop / relic / event / encounter aggregate statistics
- expose counts and confidence signals instead of silent heuristics

### 3. Coverage and quality reporting

- report build ids, characters, ascension bands, and win / loss coverage
- surface incomplete or partially normalized runs explicitly

### 4. Export and reuse

- write JSONL / CSV / summary outputs that later priors and benchmark code can load directly

### 5. Tests and docs

- cover normalization across representative run payload variants
- document the boundary between normalized public runs and local runtime trajectories

## Deliverables

- normalized public-run data contract and exporters
- derived strategic summary artifacts and reports
- tests and roadmap / issue documentation

## Delivered On The Current Branch

- `src/sts2_rl/data/public_run_normalized.py` adds a repo-native normalization contract for archived public runs
- `sts2-rl public-runs normalize` writes `normalized-public-runs.jsonl` / CSV plus strategic card, shop, event, relic, encounter, and route summaries
- `sts2-rl public-runs normalized-summary` exposes coverage and histogram diagnostics without reparsing raw archive payloads manually
- normalized records preserve provenance, benchmark-slice fields, room summaries, final deck/relic coverage flags, and missing-detail fallbacks
- regression tests cover normalized artifact generation, strategic artifact emission, missing-detail tolerance, and CLI wiring
