# Issue 0.3.1-01

## Status

Completed on 2026-04-13.

## Title

Community card stats snapshot contract

## Objective

Create a canonical local artifact for external STS2 card pick-rate / buy-rate / win-delta snapshots so the repo can consume community card statistics without depending on live websites or ad hoc CSV parsing.

## Problem

Community sites and local stats mods expose useful card-level signals, but the repo had no stable contract for importing them. Without a typed snapshot layer, every downstream use would depend on one-off parsing, weak lineage, and unstable assumptions about source columns.

## In Scope

- typed community card stat record schema
- local import pipeline from CSV / JSONL / JSON
- canonical JSONL / CSV exports plus summary artifact
- CLI import and summary commands
- roadmap / issue docs and regression tests

## Out Of Scope

- online scraping
- direct policy integration
- benchmark alignment metrics

## Dependencies

- existing artifact conventions in `src/sts2_rl/data/`
- CLI summary patterns already used for datasets and shadow artifacts

## Affected Areas

- `src/sts2_rl/data/`
- `src/sts2_rl/cli.py`
- `tests/`
- `docs/`

## Acceptance Criteria

- the repo can import community card stats from a local CSV, JSONL, or JSON payload with no custom script
- canonical records preserve source identity, snapshot date, card identity, context slice, and rate/count fields
- missing pick/buy rates are derived from counts when possible
- summary artifacts expose source, character, source-type, and rate histograms / stats
- tests cover import, loading, summary generation, and CLI execution

## Implementation Breakdown

### 1. Typed record contract

- add a `CommunityCardStatRecord` model with stable filenames and helper loaders
- validate snapshot dates, optional counts, and bounded rate fields

### 2. Import pipeline

- support CSV, JSONL, and JSON import payloads
- allow snapshot metadata overrides from CLI flags
- write canonical JSONL, CSV, and summary artifacts

### 3. Reporting

- report source histograms, card counts, source-type coverage, and top cards by pick-rate / buy-rate / win-delta

### 4. CLI integration

- add `sts2-rl community import`
- add `sts2-rl community summary`

### 5. Tests and docs

- cover count-to-rate derivation and embedded JSON metadata
- update roadmap and README with the new artifact flow

## Deliverables

- `src/sts2_rl/data/community.py`
- `sts2-rl community import` and `sts2-rl community summary`
- regression tests for community snapshot import and summary flows
- `v0.3.1` roadmap and issue documentation
