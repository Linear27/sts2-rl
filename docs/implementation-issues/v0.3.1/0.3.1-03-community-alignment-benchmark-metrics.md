# Issue 0.3.1-03

## Status

Completed on 2026-04-13.

## Title

Community alignment benchmark metrics

## Objective

Close the loop on imported community card statistics by measuring how benchmarked policies align with those priors at decision time.

## Problem

`0.3.1-02` made community priors visible inside reward, selection, and shop scoring, but the benchmark layer still could not quantify whether a policy was following or fighting those imported signals. That left imported priors as an audit-only feature: useful for log inspection, but not measurable across eval suites, checkpoint comparisons, or suite-level reports.

## In Scope

- extracting community-prior decision traces from benchmark `combat-eval.jsonl` step records
- eval-case alignment summaries for top-choice match, weighted agreement, coverage, and regret
- compare-case deltas for baseline vs candidate alignment behavior
- suite-level aggregation over eval and compare case outputs
- regression tests and docs

## Out Of Scope

- changing live policy behavior beyond the already-landed prior integration
- scraping or refreshing community sources during benchmark execution
- using community alignment as the benchmark primary metric

## Dependencies

- `0.3.1-01` community snapshot contract
- `0.3.1-02` policy-side community prior decision metadata
- existing benchmark suite raw log capture for eval / compare cases

## Affected Areas

- `src/sts2_rl/train/benchmark_suite.py`
- `tests/test_train_benchmark_suite.py`
- `docs/`
- `README.md`

## Acceptance Criteria

- eval benchmark cases expose structured community-alignment summaries sourced from raw step logs rather than reconstructed coarse summaries
- compare benchmark cases expose baseline, candidate, and delta alignment metrics over paired iterations
- suite summaries aggregate community alignment across eval and compare cases
- tests prove reward / shop / selection traces are consumed correctly and that compare deltas are reported

## Implementation Breakdown

### 1. Raw trace extraction

- parse `decision_metadata.community_prior` from `combat-eval.jsonl` step records
- normalize selected choice, ranked candidates, best prior candidate, and regret payloads into a stable benchmark trace format

### 2. Eval summaries

- report eligible decision count, aligned decision count, top-choice match rate, weighted top-choice match rate, opportunity coverage, selected-prior availability, and score / regret statistics
- expose per-domain histograms and per-domain alignment summaries

### 3. Compare deltas

- pair baseline and candidate iterations by iteration index
- compute delta metrics for top-choice match, weighted agreement, coverage, selected-prior availability, selected score bonus, and regret
- keep the existing benchmark primary metric unchanged while surfacing alignment deltas as secondary metrics

### 4. Suite aggregation

- aggregate eval-case alignment summaries at suite level
- aggregate compare baseline / candidate summaries and comparison deltas at suite level

### 5. Tests and docs

- add eval-case tests over raw reward / shop / selection logs
- add compare-case tests over baseline vs candidate alignment deltas
- update roadmap and README summaries

## Deliverables

- benchmark-suite community alignment extraction and summary helpers
- eval and compare case payloads with `community_alignment` sections
- suite summary `community_alignment` aggregation
- regression tests covering eval and compare alignment metrics
- roadmap / issue / README documentation for the completed capability
