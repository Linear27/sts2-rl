# Issue 0.3.1-02

## Status

Completed on 2026-04-13.

## Title

Reward and shop prior integration

## Objective

Inject imported community card statistics into policy-pack reward, selection, and shop scoring as explicit external priors, while keeping the repo auditable and local-first.

## Problem

`0.3.1-01` created canonical local card-stat artifacts, but the live policy stack still ignored them. Without explicit integration, imported community snapshots were dead data: reward picks, shop buys, and remove / upgrade choices could not consume the new signal, and live evaluation / benchmark surfaces had no way to configure or audit prior usage.

## In Scope

- runtime community-prior scoring module over imported `community_card_stats` artifacts
- reward-card, selection, and shop-card scoring integration inside the policy pack
- decision-trace metadata showing selected and ranked prior contributions
- collect / train / eval / runtime-job / benchmark-suite config plumbing
- regression tests and docs

## Out Of Scope

- scraping live community websites
- treating community rates as hard labels
- alignment metrics and benchmark deltas against community decisions

## Dependencies

- `0.3.1-01` community snapshot contract
- existing policy-pack scoring and predictor-guidance trace format

## Affected Areas

- `src/sts2_rl/collect/`
- `src/sts2_rl/train/`
- `src/sts2_rl/runtime/`
- `src/sts2_rl/cli.py`
- `tests/`
- `docs/`

## Acceptance Criteria

- imported community card stats can influence reward picks, selection remove / upgrade decisions, and shop-card buys through explicit config
- prior matching respects runtime context such as character, ascension, act, and floor-band slices when available
- prior bonuses are confidence-scaled rather than blindly copied from community rates
- live eval, collect, runtime-job, and benchmark-suite entrypoints can pass prior config without hidden global state
- decision traces expose selected and ranked prior payloads so audits can explain why a card moved up or down

## Implementation Breakdown

### 1. Runtime prior module

- add a dedicated `community_prior` module in `collect/`
- load canonical imported records and index them by card id
- match records by domain plus runtime context
- compute confidence-scaled score bonuses from pick/buy rates and win-delta

### 2. Policy-pack scoring integration

- apply reward priors to reward-card and reward-like selection picks
- apply inverse priors to remove flows and smaller positive priors to upgrade flows
- apply shop priors to shop-card buys without affecting relic / potion scoring

### 3. Auditability

- attach community-prior payloads to guided candidates
- surface selected and ranked community-prior traces alongside predictor traces

### 4. Config propagation

- extend collect/train/eval codepaths to accept `CommunityPriorRuntimeConfig`
- add runtime-job and benchmark-suite manifest specs for community prior config
- expose CLI flags for the live commands that execute policy-pack control

### 5. Tests and docs

- add direct prior-module tests
- add policy integration tests that prove reward/shop/remove decisions can flip because of priors
- add manifest tests for benchmark-suite and runtime-job path resolution / pass-through
- update roadmap and README usage examples

## Deliverables

- `src/sts2_rl/collect/community_prior.py`
- policy-pack reward / selection / shop integration with decision-trace metadata
- runtime-job and benchmark-suite community prior config support
- CLI support for `--community-prior-source-path` on live policy-pack consumers
- regression tests for source scoring, policy integration, manifest resolution, and benchmark propagation
