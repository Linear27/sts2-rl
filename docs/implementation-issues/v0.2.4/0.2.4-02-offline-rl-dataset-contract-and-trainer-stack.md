# Issue 0.2.4-02

## Title

Offline RL dataset contract and trainer stack

## Objective

Turn trajectory corpora into stable offline RL datasets and train at least one practical discrete offline RL baseline that can later be fine-tuned online without changing the surrounding artifact model again.

## Problem

The repo already records trajectories and can train online from live interaction, but it still cannot learn from the large amount of static data it is collecting. Without an explicit offline RL contract, each algorithm would need custom replay conversion, custom metadata, and custom evaluation glue.

## In Scope

- transition-level and episode-level offline RL dataset views
- normalization statistics and feature snapshots tied to dataset lineage
- at least one practical discrete offline RL baseline, with discrete CQL as the default first implementation
- offline training summaries plus benchmark-compatible evaluation outputs
- compatibility with later offline-to-online warm start or fine-tuning

## Out Of Scope

- broad algorithm zoo coverage in the same issue
- distributed multi-GPU training infrastructure
- simulator-only or headless training paths

## Dependencies

- dataset manifests from `0.2.3-01`
- benchmark suites from `0.2.3-02`
- behavior-cloning artifact and evaluation contract from `0.2.4-01`

## Affected Areas

- `src/sts2_rl/data/`
- `src/sts2_rl/train/`
- `src/sts2_rl/eval/`
- `src/sts2_rl/cli.py`
- `configs/`
- `tests/`
- `artifacts/`

## Acceptance Criteria

- offline RL datasets can be materialized deterministically from manifest-built sources
- the repo can train a discrete offline RL baseline from those datasets with stable summaries and checkpoints
- offline-trained checkpoints can be evaluated through the same benchmark flow used by online and BC policies
- dataset summaries include transition counts, return distributions, and source lineage needed for experiment comparison
- tests cover dataset conversion, trainer config validation, and offline evaluation output contracts

## Implementation Breakdown

### 1. Offline dataset contract

- define explicit episode and transition records for offline RL consumption instead of reusing raw trajectory JSONL directly
- persist reward, done, truncation, action-mask, and stage metadata in a shape that can feed Q-learning style objectives
- snapshot normalization statistics and feature schema versions alongside the materialized dataset

### 2. Dataset builder and inspection tools

- add builders that derive offline RL datasets from registered trajectory corpora with deterministic ordering
- add summary and inspection commands for return distributions, horizon statistics, and action-support coverage
- support dataset filtering by stage, character, floor band, and source quality so training does not rely on manual scripting

### 3. First offline RL trainer

- implement a practical discrete offline RL baseline with conservative regularization, using discrete CQL as the initial target
- reuse the repo's masked action expansion and checkpoint metadata conventions
- support validation checkpoints, early stopping signals, and best-checkpoint selection from offline metrics plus live benchmark results

### 4. Evaluation and warm-start compatibility

- add offline evaluation summaries that report Bellman-style losses, support coverage, and held-out return proxies where applicable
- add export hooks so offline-trained checkpoints can seed later online training without ad hoc conversion
- make benchmark outputs directly comparable with BC, DQN, and heuristic baselines

### 5. Tests

- test episode-to-transition conversion and statistics summaries
- test deterministic dataset materialization from the same manifest
- test trainer config validation, checkpoint metadata, and resume behavior
- test benchmark-facing summary compatibility for offline-trained checkpoints

## Deliverables

- offline RL dataset schema and builder
- discrete CQL trainer and config templates
- offline evaluation and checkpoint export flow
- dataset inspection summaries
- automated tests for conversion, training, and evaluation wiring
