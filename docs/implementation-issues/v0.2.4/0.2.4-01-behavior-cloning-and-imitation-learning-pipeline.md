# Issue 0.2.4-01

## Title

Behavior cloning and imitation-learning pipeline

## Objective

Train and evaluate supervised policy checkpoints directly from manifest-built trajectory datasets so the repo has a first-class imitation-learning path instead of heuristic-only collection plus RL-only training.

## Problem

The repo can already collect trajectories, normalize runtime state, benchmark checkpoints, and build deterministic datasets. What it still lacks is a repo-native learner that consumes those datasets and emits policy checkpoints under the same evaluation and promotion rules as the rest of the stack.

## In Scope

- supervised discrete-policy training from trajectory datasets
- action-mask-aware policy heads for combat and other staged decisions
- sample weighting and dataset filtering hooks for high-quality demonstrations
- held-out evaluation, confusion/ranking summaries, and live rollout evaluation
- checkpoint export and metadata that fit the existing artifact conventions

## Out Of Scope

- DAgger or online relabeling loops
- very large sequence-model training infrastructure
- offline RL objectives beyond pure supervised imitation

## Dependencies

- dataset registry and reproducible builds from `0.2.3-01`
- benchmark suites and statistical reporting from `0.2.3-02`
- policy-pack decision interfaces from `0.2.3-03`

## Affected Areas

- `src/sts2_rl/train/`
- `src/sts2_rl/data/`
- `src/sts2_rl/eval/`
- `src/sts2_rl/cli.py`
- `configs/`
- `tests/`
- `artifacts/`

## Acceptance Criteria

- a manifest-built trajectory dataset can be consumed without one-off conversion scripts
- behavior-cloning training writes checkpoints, metrics, and config snapshots under a stable artifact layout
- held-out evaluation reports top-k action accuracy, loss curves, and mask-aware coverage summaries
- live rollout evaluation can compare a BC checkpoint against existing heuristic or DQN baselines
- tests cover dataset ingestion, masked loss computation, checkpoint metadata, and CLI wiring

## Implementation Breakdown

### 1. Training dataset adapter

- define a training-ready view over trajectory datasets with stable feature extraction, action labels, and decision-stage filtering
- support sample weighting by outcome quality, floor band, source policy, or explicit manifest weight
- preserve deterministic example ordering and split membership from the dataset build layer

### 2. Policy model and masked supervised objective

- add a discrete policy model family that matches the repo's candidate-action expansion and action masking rules
- train with masked cross-entropy or equivalent supervised objectives so invalid actions are never treated as negatives
- support stage-specific output heads when route, shop, reward, and combat decisions need different label spaces

### 3. Evaluation and benchmark integration

- add held-out evaluation reports with per-stage accuracy, calibration-style confidence summaries, and target-balance snapshots
- add live rollout evaluation that runs BC checkpoints through the existing benchmark suite and runtime transport
- persist comparison artifacts that can be consumed later by registry and promotion tooling

### 4. CLI, configs, and artifact contract

- add CLI commands for train, resume, evaluate, and summarize behavior-cloning runs
- add configuration templates for common demonstration sources such as heuristic-only, predictor-guided, or curated high-win datasets
- persist checkpoint metadata, feature-set hashes, dataset lineage, and benchmark references with each training run

### 5. Tests

- test dataset-to-batch conversion and deterministic ordering
- test masked supervised loss and stage-specific label handling
- test checkpoint metadata and summary generation
- test CLI-facing config validation and artifact output paths

## Deliverables

- behavior-cloning training pipeline
- masked discrete policy model and training config
- held-out and live evaluation reports
- checkpoint and summary artifact contract
- automated tests for training and evaluation wiring
