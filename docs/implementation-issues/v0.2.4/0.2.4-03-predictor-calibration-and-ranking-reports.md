# Issue 0.2.4-03

## Title

Predictor calibration and ranking reports

## Objective

Make predictor outputs measurable enough to guide promotion, ablation, and fallback decisions instead of relying on raw score inspection.

## Problem

The repo already exports combat-outcome data and integrates predictor scores into control paths, but those scores are not yet evaluated as calibrated decision support. A predictor can improve average ranking while still being poorly calibrated, unstable across floor bands, or actively misleading in the tails.

## In Scope

- calibration metrics for regression or probability-like predictor outputs
- ranking metrics for card, shop, route, and combat-choice support
- reliability summaries stratified by character, floor band, and combat type
- predictor promotion thresholds and rollback signals derived from measurable reports
- CLI and artifact outputs suitable for regression testing and experiment comparison

## Out Of Scope

- training a brand-new predictor architecture family
- UI dashboards or hosted reporting systems
- replacing live benchmark evaluation with predictor-only evaluation

## Dependencies

- predictor integration artifacts from `0.2.3-04`
- benchmark reporting from `0.2.3-02`
- experiment artifact conventions used throughout `v0.2.3`

## Affected Areas

- `src/sts2_rl/predict/`
- `src/sts2_rl/eval/`
- `src/sts2_rl/cli.py`
- `tests/`
- `artifacts/`
- `docs/`

## Acceptance Criteria

- predictor evaluation reports include calibration and ranking metrics, not just aggregate error
- reports can be stratified by scenario slices such as character, floor band, or encounter family
- predictor-guided and predictor-disabled benchmark runs can be compared through a stable report format
- promotion decisions can reference explicit pass/fail thresholds from generated artifacts
- tests cover metric computation, slice handling, and CLI-facing report summaries

## Implementation Breakdown

### 1. Calibration metrics

- add expected calibration error, bin-wise reliability summaries, and regression residual slices where applicable
- track score spread, over-confidence, and under-confidence across major runtime slices
- persist both machine-readable metrics and concise human-readable summaries

### 2. Ranking and decision-support metrics

- add pairwise or top-k ranking metrics for predictor-assisted choices such as rewards, shops, routes, and combat support
- compare predictor-guided ordering against realized outcomes or benchmark deltas instead of relying on raw score magnitude
- separate "useful ranking" from "well-calibrated confidence" in the report structure

### 3. Evaluation flows and ablations

- add report builders that compare heuristic-only, predictor-guided, and ablated control modes on the same benchmark slices
- persist per-slice summaries so later registry tooling can surface regressions without manual notebook work
- encode clear promotion and rollback criteria in report metadata

### 4. CLI and documentation

- add CLI commands for predictor calibration summary, ranking summary, and slice comparison reports
- document which reports are required before predictor updates can be promoted into default control paths
- make outputs discoverable from README and roadmap references

### 5. Tests

- test calibration binning and summary aggregation on synthetic fixtures
- test ranking-metric computation and slice stability
- test CLI-facing report generation and threshold evaluation

## Deliverables

- predictor calibration report builder
- ranking and slice-comparison summaries
- promotion-threshold metadata for predictor artifacts
- CLI support for predictor report generation
- automated tests for metrics and reporting flows
