# Issue 0.2.5-03

## Title

Boss-path-aware datasets, predictors, and benchmark suites

## Objective

Make boss and route structure first-class data features so supervised policies, offline RL, and predictor-guided control stop learning from strategically incomplete observations.

## Status

Complete on the current branch.

## Problem

Even if the live heuristic path becomes boss-aware, the current dataset and predictor features still collapse map state to immediate node types and counts. That would keep BC, offline RL, and predictor-guided routing blind to the very context the live planner depends on.

## In Scope

- extend dataset summaries and exported examples with boss and route-structure features
- add predictor features for boss-conditioned map, reward, shop, and rest decisions
- add benchmark-suite cases that compare strategic planners on fixed seeds and route-sensitive slices
- preserve deterministic split and lineage behavior for the new fields

## Out Of Scope

- a brand-new predictor model family
- hosted dashboards
- replacing existing benchmark artifact formats wholesale

## Dependencies

- `0.2.5-01` strategic-state contract
- `0.2.5-02` route planner traces
- existing dataset manifest and benchmark-suite infrastructure

## Affected Areas

- `src/sts2_rl/data/`
- `src/sts2_rl/predict/`
- `src/sts2_rl/train/benchmark_suite.py`
- `configs/datasets/`
- `configs/benchmarks/`
- `tests/`

## Acceptance Criteria

- trajectory and offline datasets include boss and route-structure summaries
- predictor feature maps include boss and path-planning fields where available
- benchmark suites can slice or group by boss, floor band, and strategic route profile
- artifact lineage remains deterministic and schema-versioned
- tests cover feature extraction, dataset summaries, and benchmark manifest handling

## Implementation Breakdown

### 1. Dataset contract upgrade

- add strategic fields such as `boss_id`, `act_index`, `reachable_rest_count`, `reachable_elite_count`, and route-shape features
- persist these fields in trajectory-step, predictor, and offline-RL datasets
- version manifests and summaries so older datasets remain readable

### 2. Predictor feature upgrade

- add boss-conditioned features and route-structure features to predictor summaries
- preserve a clear distinction between immediate frontier features and whole-path features
- expose calibration and ranking slices by boss and route profile

### 3. Benchmark-suite coverage

- add fixed-seed benchmark manifests that target strategic decision quality, not just combat outcomes
- compare legacy map heuristics vs path planner vs predictor-assisted planner
- summarize route-choice deltas and downstream run outcomes together

### 4. Training integration

- ensure BC and offline RL loaders can ingest the new strategic fields without custom forks
- document which fields are mandatory for strategic-policy experiments versus optional for old artifacts
- add dataset summaries that report strategic-field coverage and sparsity

### 5. Tests

- test strategic feature extraction and summary versioning
- test predictor feature maps for boss/path slices
- test benchmark manifests and result summaries for strategic comparisons

## Deliverables

- upgraded dataset schemas and summaries
- boss/path-aware predictor features
- strategic benchmark manifests and reports
- tests for extraction, versioning, and suite handling

## Completed Work

- upgraded dataset summaries to schema version `2` with strategic coverage, boss/act histograms, route-profile histograms, and route-plan sparsity stats
- persisted `strategic_context` into predictor examples, trajectory exports, and offline RL transitions without breaking existing loaders
- added boss/path-aware predictor features for projected route plans, planned node composition, and route reason tags
- extended predictor calibration/ranking slices and benchmark comparison summaries with boss and route-profile dimensions
- upgraded benchmark suite manifests and summaries with strategic scenario fields, case-level route-planner aggregates, and suite-level scenario histograms
- added regression coverage for dataset summaries, predictor dataset extraction, predictor reports, benchmark-suite strategic summaries, and full-suite compatibility
