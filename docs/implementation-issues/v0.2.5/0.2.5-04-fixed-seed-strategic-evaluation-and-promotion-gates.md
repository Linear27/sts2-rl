# Issue 0.2.5-04

## Status

Completed on 2026-04-13.

## Title

Fixed-seed strategic evaluation and promotion gates

## Objective

Turn boss-conditioned planning into a measurable patch-line with promotion criteria that work on the current low-parallel fixed-seed workflow.

## Problem

The repo already supports fixed-seed live runs, but strategic planning changes still lack a dedicated evaluation loop. Without seed-stable route benchmarks and promotion gates, it is too easy to generate more data while silently regressing path quality, rest timing, or boss preparation.

## In Scope

- define fixed-seed strategic suites for current benchmark seeds
- add promotion thresholds for route quality and downstream survival metrics
- compare legacy vs strategic planners on the same seeds and starts
- persist route-specific diagnostics alongside current run and combat summaries

## Out Of Scope

- broad generalization benchmarking across many random seeds
- cluster-scale orchestration changes
- replacing combat-focused evaluation metrics

## Dependencies

- `0.2.5-02` boss-conditioned route planner
- `0.2.5-03` strategic benchmark support

## Affected Areas

- `configs/benchmarks/`
- `src/sts2_rl/train/benchmark_suite.py`
- `src/sts2_rl/registry/`
- `docs/`
- `tests/`

## Acceptance Criteria

- fixed-seed strategic suites can compare route decisions and run outcomes on the same start contracts
- summaries include route-quality metrics, not only final win/loss
- promotion or rollback gates can reference route-aware thresholds
- evaluation artifacts remain compatible with the current registry and compare flows
- tests cover suite configuration and promotion-threshold logic

## Implementation Breakdown

### 1. Strategic benchmark suites

- define suite cases for legacy heuristic, strategic planner, and predictor-assisted strategic planner
- lock the same game seed, character, ascension, and progress profile across comparisons
- persist route traces and downstream run summaries in a stable artifact layout

### 2. Route-quality metrics

- add route-choice agreement, route-risk profile, rest timing, elite timing, and pre-boss readiness summaries
- keep final run outcome metrics, but do not let them hide route regressions
- expose per-floor route decisions in human-readable summaries

### 3. Promotion gates

- define threshold metadata for strategic planner promotion on the fixed-seed suite
- support rollback when a planner improves one run metric while clearly degrading route quality
- store the promotion rationale in registry or comparison artifacts

### 4. Tests

- test suite configuration validation for fixed-seed strategic cases
- test route-quality aggregation and threshold evaluation
- test registry-facing promotion payloads for strategic runs

## Deliverables

- fixed-seed strategic benchmark suites
- route-aware summary metrics
- promotion-threshold metadata for strategic planners
- tests for suite execution and threshold evaluation

## Implementation Notes

- benchmark compare cases now thread `game_run_contract` through internal evaluation paths when fixed-seed contracts or promotion gates are configured
- benchmark artifacts persist route traces, route-quality summaries, paired route comparisons, and promotion decisions
- registry benchmark suite and benchmark case inspections now expose promotion and route-aware snapshot fields
- example fixed-seed strategic suites live under `configs/benchmarks/fixed-seed-strategic-suite.example.toml`
