# Issue 0.2.5-02

Status: complete.

## Title

Boss-conditioned route planner and non-combat policy upgrade

## Objective

Replace single-node route heuristics with a bounded path planner that explicitly conditions on the current boss, upcoming rest/shop/elite structure, and deck survival needs.

## Problem

The current map policy scores only the immediate node type plus coarse run context such as HP and gold. That is not enough for Slay the Spire style planning, where route value depends on the sequence of future nodes and on the boss the deck must eventually beat. This weakness is already visible in fixed-seed live runs where most map decisions collapse to `route_to_monster_progression`.

## In Scope

- add a bounded route lookahead planner for map decisions
- score path prefixes rather than only the immediate node
- inject boss-conditioned preferences into route, reward, shop, and rest logic
- persist route-planner traces into artifacts for regression review
- support explicit ablations between legacy single-node scoring and the new planner

## Out Of Scope

- full-depth dynamic programming over the entire act
- combat simulator integration
- learning-based route selection in this issue

## Dependencies

- `0.2.5-01` strategic state contract
- existing policy-pack architecture and decision-trace artifacts

## Affected Areas

- `src/sts2_rl/collect/policy.py`
- `src/sts2_rl/data/trajectory.py`
- `src/sts2_rl/cli.py`
- `tests/`

## Acceptance Criteria

- route selection can evaluate multiple-step path prefixes instead of single nodes only
- decision traces expose the chosen path score, top alternatives, and boss-conditioned reasons
- rest, shop, and reward scoring can reference boss-conditioned strategic context
- policy-pack eval can compare legacy and boss-conditioned routing on the same seeds
- regression tests cover route cases where the best first step depends on future path structure

## Implementation Breakdown

### 1. Strategic context object

- extend run context with boss id, act index, upcoming path structure, and reachable utility counts
- compute route-planning features such as distance to next rest, elites before rest, and shops before boss
- keep the context derivation deterministic and cheap enough for live runtime use

### 2. Bounded route search

- enumerate route prefixes from the current frontier for a fixed depth or until a stop condition
- score whole prefixes with additive structural bonuses and penalties
- choose the first action from the best-ranked prefix while persisting the ranked alternatives

### 3. Boss-conditioned heuristics

- bias route preferences by boss demands, such as front-load damage, sustain, AOE pressure, or upgrade urgency
- let rest, reward, and shop decisions consume the same strategic context so they align with route intent
- define explicit fallback behavior when boss metadata is missing

### 4. Trace and ablation support

- store route planner metadata in `decision_metadata` and summary histograms
- add a policy profile or switch that disables path lookahead for clean ablations
- expose planner coverage and score distributions in summaries

### 5. Tests

- add route fixtures where immediate node value loses to better downstream structure
- add boss-conditioned fixtures where the same map should yield different choices under different bosses
- test rest/reward/shop shifts driven by strategic context

## Deliverables

- bounded route planner
- boss-conditioned non-combat scoring upgrades
- richer route decision traces
- regression tests and ablation hooks
