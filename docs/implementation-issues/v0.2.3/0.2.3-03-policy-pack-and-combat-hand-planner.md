# Issue 0.2.3-03

## Title

Policy pack and combat hand planner

## Objective

Raise the repo's non-learning baseline by splitting heuristics into explicit policy packs and adding a bounded combat hand planner for higher-quality action selection.

## Problem

The current `SimplePolicy` is serviceable, but it is still one monolithic baseline. That limits regression testing, ablations, and dataset quality, especially in combat-adjacent decisions where a short planner would outperform one-step heuristics.

## In Scope

- split the current heuristic logic into explicit policy packs or profiles
- add a bounded combat hand planner or beam/tree search action ranker
- persist planner scores and chosen-policy metadata into artifacts
- add policy regression tests and coverage summaries

## Out Of Scope

- full combat simulator
- large-scale MCTS
- replacing the DQN in this issue

## Dependencies

- benchmark reporting from `0.2.3-02`

## Affected Areas

- `src/sts2_rl/collect/policy.py`
- `src/sts2_rl/train/`
- `src/sts2_rl/data/`
- `tests/`

## Acceptance Criteria

- policy packs can be selected explicitly
- combat planner decisions are visible in trajectory or eval artifacts
- regression tests cover key route, reward, shop, rest, and combat-hand scenarios
- benchmark suites show planner-vs-baseline comparisons

## Implementation Breakdown

### 1. Policy pack split

- extract the current `SimplePolicy` into smaller policy modules or profiles
- define a stable policy selection interface so collection, eval, and benchmarks can swap policy packs cleanly
- preserve the current behavior as an explicit baseline profile

### 2. Decision trace contract

- persist which policy pack handled each decision
- keep per-stage reasons and scores in artifacts
- make planner-produced rankings inspectable instead of only storing the chosen action

### 3. Combat hand planner

- add a bounded hand planner or short-horizon search layer for combat action ranking
- keep the planner runtime-bounded so it remains usable inside the live game loop
- support planner-only and planner-assisted modes for later ablations

### 4. Regression corpus

- create a test corpus of representative observations for map, reward, shop, rest, event, and combat-hand decisions
- assert stable top choices or top-ranked sets for those observations
- wire the corpus into normal test runs

### 5. Benchmark integration

- add planner-vs-baseline benchmark cases
- expose planner coverage and score distributions in summaries

## Deliverables

- policy pack abstraction and baseline profile
- combat hand planner implementation
- richer decision trace artifacts
- regression corpus and tests
- benchmark support for planner comparisons
