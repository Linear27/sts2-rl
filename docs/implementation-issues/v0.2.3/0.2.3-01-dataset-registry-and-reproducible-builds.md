# Issue 0.2.3-01

## Title

Dataset registry and reproducible builds

## Objective

Turn raw trajectory and combat-outcome artifacts into named, reproducible datasets that can be consumed by predictors, behavior cloning, and offline RL without custom one-off scripts.

## Problem

The repo emits useful artifacts, but there is still no first-class dataset contract. That makes filtering, weighting, train/validation/test splits, and data lineage fragile across experiments.

## In Scope

- dataset manifests with schema version, source paths, filters, and split metadata
- repo-native dataset build commands for trajectory and combat-outcome sources
- reproducible train/validation/test split generation
- optional export formats for downstream training
- dataset summaries with class balance, character balance, and source lineage

## Out Of Scope

- distributed data storage
- cloud object stores
- end-to-end offline RL training in the same issue

## Dependencies

- existing trajectory and predictor export artifacts from `v0.2.1`

## Affected Areas

- `src/sts2_rl/data/`
- `src/sts2_rl/predict/`
- `src/sts2_rl/cli.py`
- `tests/`
- `data/`

## Acceptance Criteria

- datasets can be rebuilt from manifests without manual path editing
- split metadata is persisted and deterministic
- summaries expose source lineage and target balance
- tests cover manifest parsing and deterministic split generation

## Implementation Breakdown

### 1. Manifest and schema layer

- define dataset manifest models for sources, filters, split policy, feature set, and output contract
- version the dataset schema explicitly instead of inferring it from file names
- support both trajectory-derived and combat-outcome-derived dataset families

### 2. Dataset build pipeline

- add a repo-native builder that resolves source artifacts, applies filters, and materializes examples
- support deterministic splitting by seed and stable source ordering
- persist per-split counts and source lineage into the dataset summary

### 3. Filtering, weighting, and export

- add reusable filters for character, floor band, win/loss, combat scope, and artifact quality
- support sample weighting hooks for later BC and offline-RL use
- add at least one structured export path beyond raw JSONL when practical

### 4. CLI and docs

- add CLI commands for manifest validation, dataset build, and dataset summary inspection
- document dataset layout under `data/`
- make the resulting datasets discoverable from README and roadmap docs

### 5. Tests

- test deterministic split generation
- test manifest validation and schema upgrades
- test lineage and summary correctness on small synthetic artifact sets

## Deliverables

- dataset manifest schema and parser
- dataset build command and output convention
- deterministic split logic
- dataset summaries with lineage and balance histograms
- automated tests for the builder and manifest layer
