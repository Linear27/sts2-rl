# Issue 0.2.3-06

## Title

Reproducibility and divergence diagnostics

## Objective

Explain why two runs diverged instead of only reporting that they diverged.

## Problem

The repo already has replay and comparison artifacts, but diagnosis is still too shallow for sustained experimentation. A failed replay can come from start-state drift, runtime drift, policy drift, or action masking changes, and those are not yet separated cleanly.

## In Scope

- start-state fingerprints and environment metadata snapshots
- structured divergence taxonomy for replay and compare artifacts
- diagnostics that separate preparation, runtime, and policy divergence
- CLI or report views for post-run diagnosis

## Out Of Scope

- perfect determinism guarantees
- engine patching for RNG control
- full savestate rollback

## Dependencies

- replay and compare hardening from `v0.2.2`
- benchmark suites from `0.2.3-02`

## Affected Areas

- `src/sts2_rl/train/replay.py`
- `src/sts2_rl/train/compare.py`
- `src/sts2_rl/runtime/`
- `src/sts2_rl/cli.py`
- `tests/`

## Acceptance Criteria

- replay and compare artifacts expose structured divergence categories
- diagnostics include preparation and start-state fingerprints
- the CLI can summarize divergence causes without raw log inspection
- tests cover the main divergence classification paths

## Implementation Breakdown

### 1. Fingerprint and metadata capture

- add start-state fingerprints and runtime metadata snapshots to replay and compare artifacts
- persist environment details that can explain drift, such as runtime target, action set, and selected policy mode

### 2. Divergence taxonomy

- classify at least the following divergence families:
  - preparation divergence
  - start-state divergence
  - policy-choice divergence
  - runtime-transition divergence
  - action-space or masking divergence

### 3. Diagnostic report builder

- add a report layer that turns raw replay or compare artifacts into structured diagnostic summaries
- preserve machine-readable categories plus a concise human-readable explanation

### 4. CLI integration

- add a CLI command or sub-mode for summarizing divergence artifacts
- make it possible to inspect causes without opening JSONL traces manually

### 5. Tests

- test fingerprint generation stability
- test divergence classification on synthetic artifact pairs
- test CLI-facing summary payload generation

## Deliverables

- fingerprint capture layer
- structured divergence taxonomy
- diagnostic report builder
- CLI support for divergence summaries
- automated tests for classification and reporting
