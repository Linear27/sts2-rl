# Issue 0.2.3-04

## Title

Predictor integration into control and scoring

## Objective

Move the predictor branch from offline training only into runtime scoring paths that can influence collection, evaluation, and policy decisions.

## Problem

The repo can already extract and train a predictor, but that branch is still disconnected from the decision loop. Without integration, the predictor cannot improve data quality, route scoring, or combat ranking.

## In Scope

- predictor inference utilities inside the runtime policy stack
- optional predictor-guided scoring for route, reward, shop, and combat ranking
- predictor ablation modes in eval and benchmark commands
- artifact fields for predictor features, outputs, and calibration summaries

## Out Of Scope

- large model serving infrastructure
- end-to-end sequence models
- replacing every heuristic with the predictor at once

## Dependencies

- predictor bootstrap from `v0.2.1`
- dataset and benchmark work from `0.2.3-01` and `0.2.3-02`

## Affected Areas

- `src/sts2_rl/predict/`
- `src/sts2_rl/collect/`
- `src/sts2_rl/train/`
- `src/sts2_rl/cli.py`
- `tests/`

## Acceptance Criteria

- predictor-guided modes can be enabled explicitly
- eval artifacts show whether predictor scoring was active
- calibration or ranking summaries are persisted
- tests cover predictor integration paths and ablations

## Implementation Breakdown

### 1. Runtime inference adapter

- add a small predictor runtime interface that can load the current predictor artifact and score examples in-process
- isolate predictor feature extraction from the existing training-only code path

### 2. Scoring hooks

- add predictor-guided scoring hooks for route, reward, shop, and combat ranking
- make each hook optional so predictor use can be turned on or off per run
- preserve baseline heuristic-only behavior for ablations

### 3. Artifact integration

- store predictor feature snapshots or score summaries in eval and collection artifacts
- record which predictor model was active and under which mode
- add calibration or confidence summaries where possible

### 4. Eval and benchmark modes

- add CLI flags for predictor-guided eval
- support direct ablation comparisons:
  - heuristic only
  - predictor assisted
  - predictor dominant where safe

### 5. Tests

- test predictor loading and inference wiring
- test that control paths behave correctly with predictor enabled and disabled
- test artifact summaries and ablation configuration payloads

## Deliverables

- predictor runtime adapter
- predictor-guided scoring hooks
- predictor-aware eval modes
- predictor metadata in artifacts
- automated tests for integration and ablations
